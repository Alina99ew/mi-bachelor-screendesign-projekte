Kurzdokumentation einreichen

Als Teil der Prüfungsleistung werden im Modul Screendesign die Projektergebnisse als Kurzdokumentation hochgeladen. Hierzu stellen Sie alle Informationen via Pull-Request bereit. Wie das genau funktioniert, wird in diesem Video erklärt. Bitte beachten Sie, dass für das Projekt im WS 2024/25 kein Styleguide abgegeben werden muss. Bei Fragen bitte das Forum im ILU-Kurs nutzen.

Checkliste

Bitte beachten Sie folgendes:

  • Sie haben einen GitHub-Account.
  • Sie haben einen Fork von dem Repository für diese Seite in ihrem GitHub-Account angelegt.
  • Sie haben alle Artefakte in den entsprechenden Ordner (_kurzdokus/sd-2024) abgelegt und haben dabei die Nomenklatur beachtet: {nachname1}-{nachname2}-{artefakt}.{ext} (Bitte keine Sonderzeichen)
    Dazu gehören:
    • Ein Header-Bild
    • Screenshots für die Galerie
  • Alle Bilder sind nicht größer als 200 KB.
  • Sie haben eine neue Markdown Datei mit allen wichtigen Informationen und Ihrem Abstract angelegt. Sie haben auch alle Bilddateien richtig in Ihrer Datei referenziert.
  • Sie haben erfolgreich einen Pull-Request gestartet, den dann das Betreuer-Team bestätigt.
